# longtext2md 需求设计文档

> 版本: v2.0 | 日期: 2026-05-19 | 状态: 待实现

## 一、项目定义

将网课视频逐字稿（可达 6w 字）结合课程代码和课件资料，自动生成高质量、零错误、信息密度高的结构化 Markdown 笔记，附带可选思维导图。

## 二、用户画像

- 个人开发者，自学网课（多数为"老师讲 + 写代码"的标准网课场景）
- 笔记品质对标"可以回看复习的完整讲义"，不是草稿
- 对成本敏感但更重质量：单门课费用 ¥1-3 可接受，耗时 ≤ 3 分钟
- 可能同时处理 5-10 门课，需要多任务并发和队列管理

## 三、输入规格

| 输入 | 格式 | 必/可选 | 说明 |
|---|---|---|---|
| 逐字稿 | `.txt` / `.md` / 剪贴板粘贴 | **必选** | 语音转文字产物，6w 字以内 |
| 代码仓库 | 目录路径 | 可选 | 课程配套代码，多文件 |
| 课件资料 | `.docx` / `.md` / `.pdf` | 可选 | 课程 PPT/讲义，可多选 |

**降级约束**：无代码/课件时管道自动跳过 RAG 索引和代码注入，仅处理文本本身。

## 四、输出规格

| 维度 | 要求 |
|---|---|
| 格式 | 单个 `.md` 文件，标准 Markdown 兼容 |
| 内容 | 层级标题 + 正文 + 内嵌代码块 + 可跳转 TOC + 可选思维导图 |
| 代码位置 | 嵌入叙事流中"老师正在讨论这段代码"的位置，非附录式堆砌 |
| 思维导图 | 可选，Mermaid `mindmap` 格式嵌入 `.md` 末尾 |
| 中间产物 | 每个管道节点输出独立文件，存储在 `output/<task_id>/` 下 |

## 五、管道架构

### 5.1 管道 DAG

```
input/
  └── raw_transcript.txt          ← 原始逐字稿
        │
  [0.0] 纯正则噪音清洗 ────────── output/<task_id>/00_cleaned.txt
        │
  [0.1] 术语词典抽取（可选）────── output/<task_id>/01_glossary.json
        │
  [0.2] 全文错别字纠正 ────────── output/<task_id>/02_corrected.txt
        │
  [0.3] 课程全局摘要 ──────────── output/<task_id>/03_summary.json
        │
  [0.4] 话题边界检测 ──────────── output/<task_id>/04_chunks.json
        │
  [0.5] RAG索引构建（可选）────── chromadb/
        │
  [1]   全并行上下文润色 ──────── output/<task_id>/05_polished.txt
        │
  [2]   全局结构化+代码注入 ────── output/<task_id>/06_structured.md
        │
  [后]  TOC生成 + 思维导图 ────── output/<task_id>/07_final.md
```

### 5.2 管道详细步骤

**阶段 0：预处理**

| 步骤 | 名称 | 方式 | 模型 | 说明 |
|---|---|---|---|---|
| 0.0 | 纯正则噪音清洗 | 纯程序 | — | 去填充词/口头禅/多余空行，保守策略 |
| 0.1 | 术语词典抽取 | 纯程序 | — | 从代码/课件正则提取技术术语，无课件跳过 |
| 0.2 | 全文错别字纠正 | LLM | DeepSeek-V4-Flash | 1w字/块，修正技术术语音译错误 |
| 0.3 | 课程全局摘要 | LLM | DeepSeek-V4-Flash | 采样法，一次调用，N次复用 |
| 0.4 | 话题边界检测 | 纯程序 | — | 多信号融合递归切割，≤2000字/块 |
| 0.5 | RAG索引构建 | 纯程序 | — | text-embedding-3-small + ChromaDB，无课件跳过 |

**阶段 1：全并行上下文润色**

| 参数 | 值 |
|---|---|
| 模型 | DeepSeek-V4-Flash (non-thinking) |
| 上下文 | 全局摘要 + 位置标签(第N/共M块) + 前块结尾80字 + 后块开头60字 + RAG代码指纹 |
| 并行策略 | asyncio.gather，所有块同时调用 |
| 输出 | 润色后干净文本，不加标题、不插代码 |

**阶段 2：全局结构化 + 代码注入**

| 参数 | 值 |
|---|---|
| 模型 | DeepSeek-V4-Pro (thinking) |
| 输入 | 阶段1 完整干净文本 + 全部代码文件 |
| 输出 | 分层级标题 + 叙事流中精确插入代码块 |

**后处理：TOC + 思维导图**

| 步骤 | 方式 | 说明 |
|---|---|---|
| TOC 生成 | 纯程序 | 解析 `##/###/####` 标题，生成可跳转目录 |
| 思维导图 | 纯程序 | 解析标题树，生成 Mermaid `mindmap` 嵌入 `.md` |

## 六、模型分配与成本

全部使用 DeepSeek API (`https://api.deepseek.com`)。

| 步骤 | 模型 | 模式 | 窗口 | 调用次数 | 费用(6w字) |
|---|---|---|---|---|---|
| 0.2 纠错 | DeepSeek-V4-Flash | non-thinking | 1M | 3-4次 | ~¥0.10 |
| 0.3 摘要 | DeepSeek-V4-Flash | non-thinking | 1M | 1次 | ~¥0.01 |
| 阶段1 润色 | DeepSeek-V4-Flash | non-thinking | 1M | 20-30次 | ~¥0.50 |
| 阶段2 结构化 | DeepSeek-V4-Pro | thinking | 1M | 1次 | ~¥0.30 |
| **合计** | | | | | **~¥0.91** |

## 七、任务系统

### 7.1 任务模型

```
Task {
  id: "task_2026-05-19_springboot"
  name: "Spring Boot 微服务实战"        ← 用户自定义或自动提取
  status: 等待中 | 运行中 | 已完成 | 失败 | 已暂停
  inputs: {
    transcript: "path/to/transcript.txt" | "粘贴文本内容"
    code_dir: "path/to/code/" | null
    courseware: ["path/to/slides.docx"] | null
  }
  pipeline: {
    current_stage: "0.2"               ← 当前执行到哪一步
    stage_status: {                    ← 每个节点的状态
      "0.0": 完成, "0.1": 完成,
      "0.2": 运行中, "0.3": 等待, ...
    }
  }
  output_dir: "output/task_2026-05-19_springboot/"
  created_at: "2026-05-19T10:30:00"
  error_log: "..." | null
}
```

### 7.2 并发控制

| 维度 | 策略 |
|---|---|
| 最大并行任务数 | 3 个任务同时跑（可配置） |
| 全局 LLM 并发上限 | 50 个同时请求（DeepSeek 默认 50 RPM） |
| 排队机制 | 超出并行数的任务进入等待队列，前面的完成自动启动下一个 |
| 任务优先级 | 默认先进先出，支持手动拖动排序 |
| 单任务失败影响 | 不影响其他任务，各自独立 |

### 7.3 检查点与重跑

| 能力 | 说明 |
|---|---|
| 中间文件持久化 | 每个节点完成后自动保存到 `output/<task_id>/` |
| 节点状态可查 | GUI 显示每个节点：⚪等待 / 🔵运行中 / 🟢完成 / 🔴失败 |
| 单节点重跑 | 选中节点后点击“从此节点重跑”，保留上游检查点 |
| 全流程重跑 | 一键从头开始，覆盖所有中间文件 |
| 单任务暂停/恢复 | 运行中的任务可暂停，后续恢复继续 |

## 八、错误处理

| 场景 | 策略 |
|---|---|
| LLM 调用超时/异常 | 自动重试 3 次（指数退避 1s/2s/4s） |
| 3 次重试均失败 | 节点标记为 🔴 失败，管道暂停 |
| 阶段1 单块失败 | 该块保留原文写入，不影响其他块继续 |
| 用户操作 | 查看错误日志 → 手动重跑该节点 → 或跳过继续 |

## 十一、非功能需求

| 维度 | 要求 |
|---|---|
| 耗时 | 单任务全流程 ≤ 3 分钟（并行优化后） |
| 输出格式 | 单个 `.md` 文件，标准 Markdown 兼容 |
| 可恢复性 | 任意节点中断后可从该节点重跑 |
| 降级 | 无代码/课件时自动跳过 RAG + 代码注入 |
| 并发 | 最多 3 任务并行，支持排队 |
| 部署 | 本地运行，无需服务器 |
| 数据隔离 | 每个任务独立 `output/<task_id>/` 子目录 |

## 十二、不做什么（明确排除）

- ❌ 不追求多格式导出（PDF/HTML）—— 专注 `.md`
- ❌ 不在 GUI 内编辑中间文件 —— 操作面板型
- ❌ 不支持多供应商 LLM —— 全部 DeepSeek API
- ❌ 不做云端部署 —— 纯本地工具
- ❌ 不支持实时流式输出 —— 只做批量处理
- ❌ 不做 prompt 版本管理/对比 —— 留给后续迭代
